§02 · the two cores

What the two cores actually mean.

两点核心到底意味着什么。

1 · A steady framework, kept consistent

1 · 搭一套框架，并始终保持它

The Agent builds one clear, predictable structure and refuses to let it drift as work piles up — so every file lives where you expect, and a messy repo becomes orderly without being rewritten.

Agent 搭起一套清晰、可预期的结构，并在工作堆积时绝不任其走样——于是每个文件都在你预期的位置，杂乱的仓库无需重写就能变得井然。

2 · Healthy code inside the framework

2 · 结构内部的健康代码

Within that structure the code stays easy to live with: comments explain the why, interfaces are designed to be used, and pieces are modular enough to reuse instead of copy. That is the code-level standard.

在这套结构内部，代码保持好相处：注释解释为什么，接口设计得便于使用，部件模块化到可以复用而非复制。这就是代码层面的规范。

§03 · modes

Two ways to put it to work.

两种上手方式。

MODE A — build from zero
模式 A — 从零搭建

Scaffold a compliant project on the first commit.

在第一次提交时就搭出合规项目。

Generates the full skeleton and writes every new file to the standard.

生成完整骨架，并把每个新文件写到规范里。

“Start a new research project here and add a model that trains on CIFAR.”
“在这里新建一个科研项目，并添加一个在 CIFAR 上训练的模型。”
MODE B — tidy an existing repo
模式 B — 整理已有仓库

Reshape a messy repo without breaking it.

整理杂乱仓库，且不破坏它。

Reorganizes by moving & renaming — never arbitrary deletion. Code with no target folder stays in root.

通过移动与重命名来整理——绝不随意删除。无归属文件夹的代码留在根目录。

“Organize this repo: move code into folders, turn training flags into configs, keep naming consistent.”
“整理这个仓库：把代码归入文件夹，把训练参数改成配置，保持命名统一。”
§04 · install

Point your agent at the spec.

让智能体指向这份规范。

# keep a stable copy, then point your agent at the skill's main file# 保留稳定副本，再让智能体指向 skill 的主文件
mkdir -p ~/ai-skills
git clone https://github.com/SaltGardenia/research-code-skill.git \

# Claude Code — wrap it as a subagent# Claude Code —— 包成 subagent

Keep a stable copy of the skill, then point your agent at its main file. The quality checker runs automatically — you never run it by hand.

保留一份稳定的 skill 副本，再让你的智能体指向它的主文件。质量检查器自动运行，无需手动执行。
